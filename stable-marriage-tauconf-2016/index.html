Akamai - The stable marriage problem

Map units (IP address prefix, traffic class) and clusters instead of men and women

Map units rank clusters by performance: latency, packet loss, throughput, etc.

Clusters rank map units by cluster contract terms, IP addresses' locality to the autonomous system in which the cluster is deployed, etc.

Akamai - Demand and capacity

Each map unit is associated with demand - the amount of content accesses that clients from that map unit are likely to generate; can vary rapidly in real-time as clients access more or less content

Each cluster has a notion of capacity that represents the maximum demand that the cluster may serve; capacity is dictated by processing power, memory and disk capacity, etc.

Akamai - algorithm extensions

Unequal number of clusters and map units - tens of millions of map units versus thousands of clusters

Partial preference lists - map unit in Boston may never need to rank a cluster in Tokyo

Modeling integral demands and capacities - express demand and capacity as arbitrary integers

Akamai - Challenges

  • Complexity and scale
  • Time to solve - re-compute every 10-30 seconds
  • Demand and capacity estimation
  • Incremental and persistent allocation
